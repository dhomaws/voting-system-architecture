# Voting System Architecture & Analysis (TM354 Project)

This project focuses on the software engineering, requirement analysis, and system design for a secure Electronic Voting System designed for parliamentary elections. It was developed as part of the TM354 course coursework.

## Project Scope & Deliverables:
- **Requirements Engineering:** Identified and documented key Functional and Non-Functional requirements (Security, Performance, and Availability) using the Volere template.
- **Use Case Modeling:** Designed system boundaries with actors (`Admin`, `Voter`, `User`) and defined relationships using `<<include>>` and `<<extend>>` stereotypes.
- **Behavioral Modeling:** Created textual descriptions and Activity Diagrams to map out the full "Cast Vote" success scenario and its exceptions.
- **Structural Design:** Developed Class and Object diagrams to handle domains like `Voter`, `Constituency`, `Ballot`, and `Candidate`.
- **Dynamic Interaction:** Designed a Sequence Diagram representing object interactions and lifelines during the vote-casting process.

## Design Tools & Concepts:
- UML Modeling (Use Case, Activity, Class, Sequence Diagrams)
- Object-Oriented Analysis and Design (OOAD)
- Requirements Verification and Specifications
